DNS 快取

DNS 快取（DNS Cache）是暫時儲存 DNS 查詢結果的機制，用於加快後續相同域名的解析速度並減少網路流量。快取可以存在於多個層級，從瀏覽器到作業系統，再到 DNS 解析器。

快取層級：
- 瀏覽器快取：儲存最近訪問網站的 DNS 記錄，通常保留幾分鐘
- 作業系統快取：系統層級的 DNS 快取，所有應用程式共享
- 路由器快取：家用路由器可能快取常用域名
- ISP 解析器快取：大規模快取，服務數千用戶

TTL 機制：
每條 DNS 記錄都有 TTL（Time To Live）值，指定記錄可以被快取的時間。例如，TTL 3600 表示記錄可以快取 1 小時。TTL 到期後，快取記錄失效，需要重新查詢。

實際應用範例：
訪問 www.google.com 時，第一次查詢可能需要 100 毫秒。後續訪問從快取讀取，只需 1 毫秒。企業內部 DNS 伺服器快取外部域名查詢，減少對外流量和延遲。

管理與清除：
Windows 使用 ipconfig /flushdns 清除快取。Linux 使用 systemd-resolve --flush-caches。瀏覽器通常在設定中提供清除選項。當 DNS 記錄變更時，可能需要等待 TTL 時間讓快取自然過期，或手動清除快取。

安全考量：
DNS 快取中毒（Cache Poisoning）是嚴重的安全威脅，攻擊者植入假記錄導致使用者訪問惡意網站。DNSSEC 可以防止這類攻擊。
