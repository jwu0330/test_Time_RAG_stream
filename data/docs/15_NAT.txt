NAT

NAT（Network Address Translation，網路位址轉譯）是一種將私有 IP 位址轉換為公有 IP 位址的技術，讓多個內部設備可以共享單一或少量公有 IP 位址存取網際網路。NAT 是解決 IPv4 位址短缺最重要的技術之一。

基本運作原理：
當內部設備發送封包到網際網路時，NAT 設備（通常是路由器）將來源 IP 位址從私有位址改為公有位址，並記錄在 NAT 表中。當回應封包返回時，NAT 設備查詢表格，將目的位址轉回原始的私有位址。

NAT 類型：
- Static NAT：一對一對映，每個私有位址對應固定的公有位址
- Dynamic NAT：從位址池動態分配公有位址
- PAT（Port Address Translation）：最常見，使用不同埠號區分連線

實際應用範例：
家用路由器使用 NAT 讓所有家庭設備（電腦、手機、平板）共享 ISP 提供的單一公有 IP。企業網路使用 NAT 保護內部網路架構，外部無法直接存取內部設備。

技術影響：
NAT 提供了額外的安全層，但也破壞了網際網路的端對端連線模型。某些協定（如 FTP、SIP）需要 NAT 穿透技術才能正常運作。
