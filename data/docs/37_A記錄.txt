A 記錄

A 記錄（Address Record）是 DNS 中最基本和最常用的記錄類型，用於將域名對映到 IPv4 位址。A 記錄是網站和網路服務運作的基礎，讓使用者能夠透過域名存取伺服器。

記錄格式：
```
example.com.    3600    IN    A    93.184.216.34
```
- example.com：域名
- 3600：TTL（秒）
- IN：網際網路類別
- A：記錄類型
- 93.184.216.34：IPv4 位址

實際應用範例：
網站 www.example.com 的 A 記錄指向 Web 伺服器的 IP 位址。郵件伺服器 mail.example.com 的 A 記錄指向郵件伺服器 IP。FTP 伺服器 ftp.example.com 的 A 記錄指向檔案伺服器 IP。

多個 A 記錄：
一個域名可以有多個 A 記錄，指向不同的 IP 位址。這稱為 DNS 輪詢（Round Robin），可以實現簡單的負載平衡。DNS 伺服器會輪流返回不同的 IP 位址，分散流量到多台伺服器。

管理考量：
A 記錄的 TTL 設定影響變更生效時間。較短的 TTL（如 300 秒）允許快速變更，但增加 DNS 查詢量。較長的 TTL（如 86400 秒）減少查詢量，但變更需要更長時間生效。變更 A 記錄前，建議先降低 TTL，變更完成後再恢復。
